Client

You can download the source code for the client here: https://github.com/MTUCappuccino/Helium

To build, open a command line and type ant. You should be able to launch the executable jar like any other java application.

Click the plus button (or "Add Server"). If the server is running, enter the address and port of the server, as well as a handle and a password (if required). Press connect. You should now be able to send messages and receive messages.

Type a message in the message window and press enter to send.

Click the image button to open a file explorer to find and image to send. (Not fully implemented yet.)

The sidebar is a work in progress and doesn't fully respresent the data of the current server.

Server

You can download the source code for the server here: https://github.com/MTUCappuccino/Helium_Server

To build, open a command line and type ant. You need to launch the server from the command line by using java -jar Helium_Server.jar.

Commands:

make - starts a new server. The server will ask the following questions:

  • Port to open
  • Server Name
  • Password entry?(Y/N)
  • Password (if you answered yes above)
  • Preset Server color (Program will prompt options)
  • HEX Color code (if you answered no above)
  • Custom Icon URL(Y/N)
  • URL if you answered yes above

shutdown - stops the server.

help - shows help text

edit - change configurable options (name, color, icon)

Known issues:

  • Messages containing a \n character may behave unpredicatably and crash the server.
  • The server may crash if a client disconnects during the initial handshake.

Central Server

The code for the central server is located here: https://github.com/MTUCappuccino/Helium_Central_Server

You can run the central server as described above, but it will not be at the IP address that the client expects it to be at. Currently, we have no dedicated system that runs the central server full time. This will be fixed in the next sprint.
