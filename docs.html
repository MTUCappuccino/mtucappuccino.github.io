Client

You can download the source code for the client here: https://github.com/MTUCappuccino/Helium

To build, open a command line and type ant. You should be able to launch the executable jar like any other java application.

Click the plus button (or "Add Server"). If the server is running, enter the address and port of the server, as well as a handle and a password (if required). Press connect. You should now be able to send messages and receive messages. You can also add a server by code if the server is registered with the central server. Contact the server vendor for a join code.

Type a message in the message window and press enter to send.

Click the image button to open a file explorer to find and image to send.

You can click the sidebar button to view server information as well as the leave server button. The Edit handle button is not fully implemented server-side. This also is where you can change notification settings.

To send an at-mention, type "@" followed by the handle of the user you want to mention. They will then be notified directly if they have at-mention notifications enabled

Clicking the Bookmarks button will open a list of all the bookmarked messages. Clicking the button again will go back to all of the messages. To bookmark a message, right click it and then click Bookmark

Known issues:

  • If a server does not require a handle, occasionally the client freaks out.
  • Sometimes if you attempt to rejoin a server or another server after failing to join a prior server, the password/handle screens will not appear. Restart the client to fix.

Server

You can download the source code for the server here: https://github.com/MTUCappuccino/Helium_Server

To build, open a command line and type ant. You need to launch the server from the command line by using java -jar Helium_Server.jar.

Commands:

make - starts a new server. The server will ask the following questions:

  • Port to open
  • Server Name
  • Password entry? (Y/N)
  • Generate random password? (Y/N)
  • Password (if you answered yes above)
  • Preset Server color (Program will prompt options)
  • HEX Color code (if you answered no above)
  • Custom Icon URL(Y/N)
  • URL if you answered yes above

edit - change one of the options set in make such as name, color, icon, password, etc. The program will prompt for valid inputs and walk you through the process.

kick - will ask for a handle and kick the user with that handle

names - lists all the names of users currently online

register - register this server with the central server. Currently not reversable

short code - returns the 5 character code obtained in the register command

stats - prints some stats about the server

shutdown - stops the server.

help - shows help text

Known issues:

  • Messages containing a \n character may behave unpredicatably and crash the server.
  • The server may crash if a client disconnects during the initial handshake.
  • Sometimes the server crashes if you enter the wrong password several times

Central Server

The code for the central server is located here: https://github.com/MTUCappuccino/Helium_Central_Server

You can run the central server as described above, but it will not be at the IP address that the client and server expect it to be at.

We have a raspberry pi running the central server waiting for client and server connections. The servers can register with this server and will be provided with a 5 character code. This same server also services clients who wish to translate the code into a usable server URL and port.

The central server was also intended to maintain a listing of publically available servers, but we did not get to implementing this feature.
